Perplexity · SearchGPT · AI Overview · Claude · Gemini가 답변에 발췌하는 콘텐츠는 공통점이 있습니다.
AI가 인용하기 좋아하는 5대 원리와, GEO Score AI 11개 KPI별 점수 향상 실전 팁을 정리했습니다.

★홈페이지에서 AI가 인용을 좋아하는 5대 원리

1

정적 HTML + 즉시 노출

AI 봇 대부분(GPTBot · ClaudeBot · PerplexityBot)은 JavaScript를 실행하지 않습니다. 콘텐츠가 raw HTML에 그대로 박혀 있어야 인식됩니다.

좋은 예워드프레스 · Hugo · Astro · Jekyll · Next.js SSR
나쁜 예React SPA · 네이버블로그 · Wix · 단순 클라이언트 렌더링
2

JSON-LD Schema가 raw HTML에 박혀 있음

Organization · WebSite · BreadcrumbList · FAQPage · Article Schema가 페이지 소스에 직접 들어가야 AI가 메타데이터를 이해합니다. JS로 주입된 Schema는 자주 누락됩니다.

3

E-E-A-T 신호 가시성

실명 운영자 · 자격 · 경력 · 연락처 · 사업자번호 · 회사 연혁 — 6대 신뢰 신호가 홈페이지 표면에 노출되어야 AI가 "신뢰할 만한 출처"로 가산.

4

구체적 데이터·수치 명시

"10년 경력 · 1,247건 시술 · 평균 만족도 4.8/5.0" 같은 구체 수치가 본문·메타에 등장해야 AI 답변에 포함될 가능성이 높아집니다.

5

CTA·연락처가 명확

"상담 예약" · "전화 02-1234-5678" · "예약 폼" 같은 행동 유도 요소가 페이지마다 명확해야 AI가 답변 끝에 출처로 추천하기 쉽습니다.

1AI 봇 접근 (가중치 12)

왜 중요: AI 봇이 사이트 진입을 차단당하면 콘텐츠가 학습·인용 풀에 들어가지 못합니다.
점수 = robots.txt에서 GPTBot · ClaudeBot · PerplexityBot · Google-Extended · Bingbot 등 7대 봇 허용율
  • robots.txt에 7대 AI 봇 모두 Allow 명시: User-agent: GPTBot + Allow: /
  • 특정 봇만 막고 싶으면 Disallow: /private/로 경로만 제한 (전체 차단 X)
  • Cloudflare · Vercel WAF 봇 차단 설정 점검 — 기본값으로 AI 봇 차단되는 경우 多
  • 방화벽/CDN 레벨에서 IP 차단 풀기
/robots.txt 접속 후 7대 봇 모두 Allow인지 확인

2Sitemap 상태 (가중치 9)

왜 중요: AI 크롤러가 사이트 전체 페이지를 발견하는 진입점.
점수 = sitemap.xml URL 개수 (5,000+ 풍부 / 1,000+ 양호 / 100+ 충분)
  • sitemap.xml 자동 생성: 워드프레스 → Yoast SEO · Rank Math, 정적 사이트 → 빌드 시 자동
  • robots.txt 마지막 줄에 Sitemap: https://example.com/sitemap.xml 명시
  • 대형 사이트(10,000+)는 sitemap-index.xml로 분할
  • 각 URL에 <lastmod> 포함 — 신선도 신호
  • Google Search Console · Bing Webmaster에 sitemap 제출
/sitemap.xml 직접 열어 URL 100건 이상 보이는가

3검색 색인 (가중치 11)

왜 중요: 색인 안 된 페이지는 AI 답변에 절대 등장 못 함.
점수 = sitemap × 0.6 + 내부 링크 다양성 × 1.2 (추정 색인 페이지 수)
  • 모든 페이지에 내부 링크 3개 이상 — orphan 페이지 제거
  • 핵심 페이지는 푸터·사이드바에서 링크
  • 새 콘텐츠는 메인 페이지에서 직접 링크 → 빠른 색인
  • noindex 메타 점검 — 실수로 색인 차단된 페이지 확인
  • 중복 콘텐츠는 canonical 태그로 정리
Google에서 site:example.com 검색 시 충분히 많은 결과 나오는가

4구조화 데이터 Schema (가중치 12)

왜 중요: AI 인용에서 가장 강력한 신호. JSON-LD가 메타데이터를 명시적으로 제공.
점수 = JSON-LD 적용 + 5대 Schema 타입 (Organization · WebSite · BreadcrumbList · FAQPage · LocalBusiness)
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "메타비즈101",
  "url": "https://example.com",
  "logo": "https://example.com/logo.png",
  "founder": { "@type": "Person", "name": "심재우" },
  "sameAs": ["https://blog.naver.com/...", "https://youtube.com/..."]
}
</script>
  • 홈페이지: Organization + WebSite 필수
  • 지역 사업자: LocalBusiness 추가 (주소·영업시간·전화)
  • FAQ 페이지: FAQPage — AI Overview에 직접 발췌
  • 모든 페이지 상단: BreadcrumbList
  • Schema 검증: validator.schema.org
페이지 소스 보기 → application/ld+json 검색 → 5개 이상 Schema 보이는가

5페이지 정보 (가중치 8)

왜 중요: AI 답변 카드의 제목·요약은 메타 태그에서 추출.
점수 = 7대 메타 신호 (title · description · OG title · OG description · OG image · canonical · viewport)
  • <title>: 50~60자 + 핵심 키워드 + 브랜드명
  • meta description: 130~160자 + 차별점 명확히
  • og:title · og:description · og:image 별도 작성 (SNS 공유 카드용)
  • og:image는 1200×630, 핵심 메시지 텍스트 포함
  • canonical로 URL 정규화 — 중복 노출 방지
  • viewport 모바일 반응형 필수
7개 메타 태그 모두 채워져 있나

6외부 권위 (가중치 9)

왜 중요: 다른 사이트가 우리를 인용·언급할수록 AI가 권위 있는 출처로 인식.
  • SNS 5개 이상 활성 운영: YouTube · Instagram · Facebook · X(트위터) · LinkedIn 등
  • 홈페이지에서 SNS 채널로 양방향 링크
  • 언론 기사 게재 → "보도된 매체 로고" 섹션에 노출
  • 업계 협회·인증 로고 표시 (가능한 경우)
  • 고객사·파트너 로고 표시
  • Wikipedia 등재 추진 — 가장 강력한 권위 신호
홈페이지 푸터·사이드바에 SNS 5개 이상 + 외부 인정 로고 보이는가

7E-E-A-T 페이지 (가중치 9)

왜 중요: AI는 신뢰할 수 있는 정보 출처를 우선 인용. 실명·자격·경력·연락처가 핵심.
점수 = 6대 신뢰 신호 (저자/자격/연혁/연락처/회사명/사업자) 합산
  • /about · /소개 페이지: 운영자 실명 · 사진 · 약력 · 자격증
  • "전문의 · 박사 · 석사 · 인증 · MD · PhD" 같은 자격 키워드 노출
  • 회사 연혁: "2015년 설립 · 11년차 운영"
  • 전화 · 이메일 · 주소 명확히 (footer)
  • 사업자등록번호 · 통신판매신고번호 (전자상거래법)
  • 고객 후기 · 평점 · 사례 페이지
/about 페이지 클릭하면 실명 + 자격 + 경력 보이는가

8CMS 자율성 (가중치 8)

왜 중요: 자율형 CMS(워드프레스 등)는 Schema·robots·sitemap 자유 수정 가능. 임대형(네이버·Wix)은 제약.
자율형 = 90점 / 하이브리드 = 60점 / 임대형 = 30점
  • 자율형 추천: 워드프레스 · Hugo · Astro · Next.js · Jekyll
  • 임대형 비추천: 네이버블로그 · Wix · Cafe24 · 카페 · imweb (Schema 자유 수정 X)
  • 이미 임대형이라면 → 핵심 콘텐츠는 별도 자율형 도메인에 미러링
  • 도메인은 자기 소유로 (서브도메인 X, 고유 도메인 ✓)
현재 CMS가 robots.txt · Schema · sitemap 모두 자유 수정 가능한가

9CTA 설계 (가중치 12)

왜 중요: 명확한 행동 유도가 있어야 AI가 답변 끝에 "여기서 상담받으세요"로 자연스럽게 추천.
  • CTA 버튼 페이지당 35개 이상 노출 (네비게이션 · 사이드바 · 본문 · 푸터)
  • CTA 문구는 구체적 동사: "상담 예약" · "무료 진단" · "30초 견적" — "더 알아보기" 같은 모호한 표현 X
  • 전화번호: tel: 링크로 클릭 가능하게
  • 이메일 폼: 이름 + 연락처 + 메시지만 (긴 폼 X)
  • 모바일 하단 고정 CTA 바 권장
  • FAQ 섹션 끝마다 CTA
홈페이지 스크롤 시 어디서든 CTA 버튼 보이는가

10모바일 성능 (가중치 10)

왜 중요: Google 모바일 우선 색인 + AI 답변 사용자 70%+ 모바일.
점수 = 4대 신호 (viewport 메타 · 반응형 CSS · 이미지 최적화 · 빠른 로딩)
  • <meta name="viewport" content="width=device-width, initial-scale=1"> 필수
  • 이미지: WebP 또는 AVIF 포맷 + lazy loading (loading="lazy")
  • Core Web Vitals 최적화: LCP < 2.5초, CLS < 0.1, FID < 100ms
  • 폰트: Pretendard 같은 한국어 최적화 폰트 + font-display: swap
  • JS 번들 분할 · 비동기 로딩
  • CDN 사용 (Cloudflare · Vercel · Netlify)
pagespeed.web.dev에서 모바일 90점 이상

11AI 인용 가능성 — 실측 (가중치 12)

왜 중요: 위 1~10번이 잘 되면 자동으로 올라가는 통합 지표. 실제 Gemini가 가상 질문에 답할 때 브랜드 인용율.
인용율 80%+ = 100점 / 50%+ = 85 / 30%+ = 65 / 15%+ = 40
  • 브랜드명 자연 반복: 본문에 5~10회 자연스럽게 등장
  • 고유 용어 정의: "GEO Score · 5C1S · AIO SEO" — 자기 사이트가 정의한 용어가 다른 곳에서 재인용될 때 출처가 됨
  • 구체 수치: "1,247건 시술 · 만족도 4.8" → AI 답변에 그대로 인용
  • FAQ 섹션 필수 + FAQPage Schema → AI Overview 직접 발췌
  • 외부 인용 표기: 통계 출처 · 논문 인용 → 권위 신호
홈페이지에서 자기 브랜드명 10회 이상, 고유 용어 1개 이상 등장하나

🎯 홈페이지 ROI 우선순위

  1. JSON-LD Schema 5종 적용 (Organization · WebSite · LocalBusiness · BreadcrumbList · FAQPage) — 단번에 +20~25점
  2. robots.txt에 7대 AI 봇 Allow — 단번에 +12점
  3. /about 페이지 + E-E-A-T 신호 6종 — +9점
  4. CTA 35개 이상 + 명확한 동사 — +12점
  5. sitemap.xml 자동 생성 + Search Console 등록 — +9점

★블로그에서 AI가 인용을 좋아하는 5대 원리

1

정의문(Definition Sentence) 구조

"X는 ~이다" 형식의 단정적 문장이 가장 자주 발췌됩니다.

좋은 예"AIO SEO는 생성형 AI 검색엔진에서 콘텐츠가 우선 인용되도록 최적화하는 전략이다."
나쁜 예"오늘은 AIO SEO에 대해 알아볼까요? 여러 가지 방법이 있는데..."

→ 모든 H2 다음 첫 문단을 정의문으로.

2

질문 → 즉답 패턴

H2를 질문 형태로, 본문 첫 줄에 답을 주면 AI가 그대로 인용합니다.

좋은 예H2: "AIO SEO와 일반 SEO는 무엇이 다른가?"
본문: "일반 SEO는 검색 순위 최적화이고, AIO SEO는 AI 답변 생성 시 출처로 인용되도록 최적화하는 것이다."
나쁜 예H2: "AIO SEO 알아보기"
본문: "안녕하세요 여러분..."

→ 글 안 H2의 50% 이상을 질문형으로.

3

권위 신호 (E-E-A-T)

Experience(경험) · Expertise(전문성) · Authoritativeness(권위) · Trustworthiness(신뢰). AI는 권위 있는 출처를 우선 인용합니다.

  • "10년간 50개 사이트 진단 결과" → Experience
  • "박사 · 전문의 · 자격증" 명시 → Expertise
  • 논문 · 기관 출처 인용 → Authoritativeness
  • 실명 · 약력 · 연락처 → Trustworthiness
4

구체적 수치·데이터

AI는 수치 · 비율 · 날짜 · 통계가 있는 문장을 발췌 우선.

좋은 예"AI 검색 인용율은 정적 HTML 사이트가 JS 사이트보다 5~10배 높다."
나쁜 예"정적 사이트가 JS 사이트보다 더 잘 인용된다."

→ 각 글에 핵심 수치 3개 이상.

5

정적 HTML + Schema

JSON-LD Schema가 raw HTML에 박혀야 인식. JS 주입 Schema는 AI 봇이 못 봅니다.

  • 워드프레스 · 티스토리 · Hugo · Astro 추천
  • 네이버블로그 · Wix · React SPA 비추천

1발행 빈도·최신성 (가중치 14)

왜 중요: AI 검색 엔진은 최신 정보를 우선 인용. 70일 정체 = 죽은 사이트.
최신성 50점 (7일내=50, 30일내=35, 90일내=20) + 30일 발행량 30점 + 90일 발행량 20점
  • 주 1회 정기 발행 절대 사수 — 짧아도 됨
  • 신규 발행 어려우면 기존 글 업데이트 (datePublished 갱신)
  • 분기별 "트렌드 리포트" 발행 — 시즌성 질문 답변에 인용
  • 연도/월을 글 제목에 포함: "2026년 AIO SEO 가이드" → "2025년" 글보다 우선
마지막 글이 7일 이내인가

2누적 글 양 (가중치 8)

왜 중요: AI 학습 데이터 풀에 들어가려면 일정 임계량 필요.
1000+ = 100 / 500+ = 92 / 200+ = 85 / 100+ = 72 / 50+ = 60
  • 200건이 1차 임계 — AI 인용 풀 안정 진입
  • 한 토픽을 50건씩 깊게 (분산보다 효과적)
  • 연재(Series) 글로 양 늘리기 — 1편씩 짧게, 연결 링크
  • 파생 콘텐츠: 1글 → Q&A · 요약 · 인포그래픽 변환으로 3~5배
1년 누적 200건 = 주 4건 페이스

3카테고리 깊이 (가중치 10)

왜 중요: 카테고리 구조 = 토픽 권위의 시각적 증명. AI가 "이 사이트는 X 전문" 인식.
카테고리 수 50점 + 카테고리당 글 수 50점 (30글/카테고리=만점)
  • 8~15개 핵심 카테고리 — 너무 많으면(50+) 분산 페널티
  • 카테고리당 최소 8건 누적 — 빈 카테고리 = 약한 신호
  • 3단계 계층: 대분류 → 중분류 → 세분류
  • 카테고리 이름은 검색어 형태: "AI 검색 최적화" > "기술 노트"
  • 사이드바에 글 수 명시: "AI SEO (47)"
가장 적은 카테고리도 8건 이상인가

4내부 링크망 (가중치 10)

왜 중요: AI는 "관련 글들이 서로 연결된" 사이트를 깊이 있다고 평가.
글당 평균 링크 수: 10+ = 100 / 5+ = 85 / 3+ = 70 / 1+ = 40
  • 글마다 관련 글 5개+ 링크 — 본문 자연스러운 인용
  • 하단 "관련 글" 박스 — 같은 카테고리 + 다른 카테고리 섞기
  • 본문 중간 앵커 텍스트는 키워드: "[AI 봇 차단 설정](...)" > "[여기](...)"
  • 주제 클러스터: 허브 글(개론) + 스포크 글(세부) 양방향 링크
  • 오래된 글에서 새 글로 링크 추가 → 신글 indexing 가속
마지막 글에 다른 글로 가는 링크 5개 이상 있나

5작성자 권위 (가중치 11)

왜 중요: AI는 익명 콘텐츠보다 실명·권위 있는 작성자를 우선 인용.
작성자 노출률 70점 + about 페이지 30점
  • 모든 글 하단에 작성자 박스 — 사진 + 실명 + 1줄 약력 + SNS
  • /about 페이지 필수 — 경력 · 자격 · 출판물 · 인터뷰
  • JSON-LD author Schema 적용
  • 작성자 페이지 별도: /author/심재우
  • 외부 인용 · 인터뷰 모음 페이지 → 권위 누적
글 하단 작성자 정보 + /about 약력 보이는가

6토픽 권위 (가중치 10)

왜 중요: AI가 "이 사이트는 X 분야 전문"이라 판단하는 근거.
최상위 카테고리 비율 40~70%=만점 / 70%+ → 75 / 25~40% → 65 / <25% → 40
  • 메인 토픽이 40~70% 차지 — "이 블로그는 X 전문" 인식
  • 너무 분산(<25%): 잡블로그 → 카테고리 통합
  • 너무 집중(>70%): 보조 토픽 1~2개 추가
  • First-party 정의: "5C1S 모델" · "GEO Score" — 내가 먼저 정의한 용어
  • 다른 사이트가 우리 용어 쓰면 → 권위 비행기 시동
메인 카테고리 글이 전체의 40~70%인가

7사용자 참여 (가중치 9)

왜 중요: AI는 댓글·공유 수가 많은 글을 "사회적 검증된" 콘텐츠로 가산.
충족 신호/4 × 100 (댓글 · 좋아요 · 공유 · 소셜 버튼)
  • 댓글 시스템 필수 (기본 + Disqus + 카카오톡 댓글)
  • 글 페이지 + 인덱스 양쪽에 좋아요/공유 버튼
  • 공유 버튼 카카오 · 페이스북 · X · 링크드인 4종 + 복사
  • 댓글 수 카운트 노출 ("댓글 23")
  • 댓글에 저자 답글 — engagement + 활성 사이트 인식
4개 신호 모두 있는가

8채널 확장 (가중치 11)

왜 중요: 멀티채널 운영자는 "실제 활동하는 권위자"로 인식.
6개 채널=100 / 5개=90 / 4개=75 / 3개=60
  • 사이드바에 SNS 아이콘 6개+: YouTube · Instagram · X · Facebook · LinkedIn · 네이버 · 티스토리 · 브런치 중 6개
  • 각 채널마다 본 블로그 링크 — 양방향 신호
  • 푸터 · about 페이지에도 SNS 링크 반복
  • 링크 텍스트: "YouTube 채널" > "여기"
  • 활성 운영 중인 채널만 — 죽은 SNS는 페널티
사이드바 · 푸터에 SNS 아이콘 6개 이상

9가독성 (가중치 7)

왜 중요: AI는 충분히 길고 잘 분리된 본문을 의미 단위로 추출하기 쉬움.
길이 60점(1500자=만점) + 단락 수 40점(10개=만점)
  • 글당 최소 1500자, 권장 3000~5000자
  • 단락마다 2~4줄 — 모바일 가독성
  • <p> 태그 사용 (10개+) — <br> 남발 금지
  • H2-H3 계층 명확히: 글당 H2 5개+ + H3 10개+
  • 리스트(<ul>/<ol>) 적극 활용 — AI 발췌 쉬움
  • TL;DR · 핵심 요약 글 상단 배치
본문 1500자+ + 단락 10개+

10블로그 Schema (가중치 10)

왜 중요: JSON-LD Schema는 AI가 글 메타데이터(저자 · 날짜 · 제목)를 정확히 파악하는 핵심.
샘플 글 적용율 80점 + 인덱스 적용 20점
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "...",
  "datePublished": "2026-04-28",
  "dateModified": "2026-04-28",
  "author": { "@type": "Person", "name": "..." },
  "image": "...",
  "publisher": { "@type": "Organization", "name": "...",
    "logo": { "@type": "ImageObject", "url": "..." } }
}
</script>
  • 모든 글 페이지에 BlogPosting 또는 Article Schema
  • 인덱스 페이지에 Blog Schema + mainEntity로 최근 글 리스트
  • FAQ 글에는 FAQPage Schema → AI Overview 직접 발췌
  • 워드프레스: Yoast SEO · Rank Math · SEOPress 자동 적용
  • 티스토리: 스킨 코드에 직접 삽입
  • 검증: validator.schema.org
글 페이지 소스에서 BlogPosting 보이는가

11AI 인용 가능성 — 실측 (가중치 12)

왜 중요: 위 1~10번이 잘 되면 자동으로 올라가는 통합 지표. Gemini가 가상 질문 6개에 답할 때 브랜드 인용율.
인용율 80%+ = 100 / 50%+ = 85 / 30%+ = 65 / 15%+ = 40
  • 브랜드명 자연 반복: 본문에 5~10회 등장 (첫 문단 · 결론 · 사례)
  • 고유 용어 만들기: "5C1S 모델" · "GEO Score AI" — 자기 정의 용어
  • 명확한 정의문: H2 + 첫 문단
  • 구체적 수치: "정적 HTML이 JS보다 5~10배 인용율 높음"
  • Q&A 구조: FAQ 섹션 + 질문형 H2 50%+
  • 외부 인용 표기: 통계 · 논문 · 언론 출처
  • CTA: 글 하단 "더 알아보기 · 문의" — AI가 출처 추천
본문에 자기 브랜드명 5회+ + 고유 용어 1개+
