1. 플랫폼 소개

GEO Score AI는 AI 검색 시대 기업의 마케팅·홍보 활동을 객관적인 수치로 진단하고, 부족한 영역을 AI가 채워주는 구조로 계약을 유도하는 플랫폼입니다.

이 플랫폼은 단순 분석 툴이 아니라, "진단 → 점수화 → 문제 인식 → 해결 제안 → 계약 전환"까지 이어지는 완전한 세일즈 퍼널 엔진입니다.

💡 핵심 철학: "문제를 보여주고, 해결을 유일하게 만들고, 계약으로 연결한다"

2. 빠른 시작

3단계 진단 절차

  1. 홈 화면 진입 → 메인 페이지의 진단 폼을 확인합니다.
  2. 정보 입력 → 기업/기관명과 홈페이지 URL을 입력합니다. 업종은 자동 감지되지만 직접 선택도 가능합니다.
  3. 진단 시작 → 약 30초 후 10대 KPI 점수, 레이더 차트, 문제 영역, 솔루션 제안이 자동 생성됩니다.
⚠️ 주의: 일부 사이트는 robots.txt 정책상 자동 크롤링을 차단할 수 있습니다. 이 경우 휴리스틱 점수가 표시됩니다.

3. 10대 KPI 가이드

GEO Score AI는 AI가 인용하고 추천하는 기준에 맞춰 재설계된 10가지 KPI를 측정합니다. 각 KPI에는 가중치(합계 100%)가 부여되어 종합 점수가 산출됩니다. ⭐ 표시된 두 KPI(AI 인용 5신호, CEP 장면 점유)는 본 플랫폼의 자체 차별점입니다.

# KPI 가중치 핵심 측정
①botAccess AI 봇 접근14%robots.txt에서 7대 AI 봇 차단 여부
②sitemapStatus Sitemap 상태10%sitemap.xml 존재·등록 URL 수
③indexExposure 검색 색인13%실제 검색 색인된 페이지 수
④structuredData 구조화 데이터12%Schema.org · FAQ JSON-LD 적용
⑤pageInfo 페이지 정보8%메타 태그 · canonical · OG 이미지
⑥contentDepth 콘텐츠 깊이10%블로그 발행 빈도 + 본문 분량/구조
⑦externalAuthority 외부 권위9%백링크 · 언론 인용 휴리스틱
⑧eeat E-E-A-T 신호8%작성자·자격·연혁·연락처 노출
⑨aiCitation AI 인용 5신호 ⭐10%ai_writing 5신호 (자체 차별점)
⑩cepScene CEP 장면 점유 ⭐6%"순간(scene)" 콘텐츠 (자체 차별점)

① AI 봇 접근 (botAccess) — 가중치 14%

AI 검색 엔진의 크롤러(GPTBot, ClaudeBot, PerplexityBot, Google-Extended 등 7대 AI 봇)가 사이트를 읽을 수 있는지 평가합니다. 사이트 루트의 robots.txt 파일에서 이들 봇이 차단되어 있다면 어떤 콘텐츠도 AI 답변에 인용될 수 없으므로 가장 높은 가중치를 가집니다. 측정: /robots.txt 파싱 후 7봇 차단 여부 매핑.

② Sitemap 상태 (sitemapStatus) — 가중치 10%

사이트의 모든 페이지를 AI·검색엔진이 효율적으로 발견하도록 안내하는 sitemap.xml 지도 파일의 유무와 품질을 평가합니다. 등록된 URL이 많고 최근 갱신일이 신선할수록 점수가 올라갑니다. 측정: /sitemap.xml 직접 fetch 후 URL 노드 개수 + lastmod 분석.

③ 검색 색인 (indexExposure) — 가중치 13%

구글·네이버에 실제 색인된 페이지 수를 측정합니다. 사이트 지도가 있어도 검색엔진이 색인하지 않으면 노출되지 않으므로 별도 KPI로 분리되어 있습니다. 측정: site: 연산자 휴리스틱 + 외부 신호 추정으로 색인 페이지 수 산출.

④ 구조화 데이터 (structuredData) — 가중치 12%

AI가 페이지 내용을 정확히 이해하도록 돕는 Schema.org 마크업(JSON-LD)의 적용도를 평가합니다. Organization, Article, FAQPage, LocalBusiness 등의 스키마가 있으면 AI 답변 인용 가능성이 크게 올라갑니다. 측정: HTML 내 application/ld+json 블록 파싱 후 스키마 종류 카운트.

⑤ 페이지 정보 (pageInfo) — 가중치 8%

각 페이지의 기본 메타 정보 - 제목 태그, 메타 설명, canonical URL, Open Graph 이미지 - 가 정확히 설정되어 있는지 평가합니다. 메타 정보가 부실하면 AI가 페이지를 요약·인용하기 어렵습니다. 측정: <title> 길이, meta description, rel=canonical, og:image 존재 여부.

⑥ 콘텐츠 깊이 (contentDepth) — 가중치 10%

블로그 등 콘텐츠 발행 빈도와 각 글의 본문 분량·구조를 측정합니다. AI는 신선하고 깊이 있는 콘텐츠를 우선 인용하므로 정기 발행이 핵심입니다. 측정: 블로그 글 개수 + 평균 글자 수 + 헤딩(H2/H3) 구조 점수.

⑦ 외부 권위 (externalAuthority) — 가중치 9%

다른 사이트에서 우리 사이트를 인용·링크하는 정도(백링크)와 언론 보도 노출을 평가합니다. 외부에서 인정받는 사이트일수록 AI가 신뢰합니다. 측정: 백링크 휴리스틱(공개 데이터) + 언론 보도 키워드 매칭.

⑧ E-E-A-T 신호 (eeat) — 가중치 8%

구글이 강조하는 4가지 신뢰도 신호 - 경험(Experience), 전문성(Expertise), 권위(Authority), 신뢰성(Trust) - 의 페이지 노출도를 평가합니다. 작성자 프로필, 자격 정보, 회사 연혁, 연락처가 명시되어야 합니다. 측정: 작성자 박스, 자격 라이선스 텍스트, 회사 연혁 페이지, 연락처 푸터 검출.

⑨ AI 인용 5신호 (aiCitation) ⭐ — 가중치 10%

본 플랫폼의 핵심 자체 차별점입니다. ai_writing 스킬의 5가지 정량 신호(질문형 H2 비율, 정의문 H2 비율, 브랜드 반복, 외부 인용, CTA 도달률)를 콘텐츠에서 직접 측정해 AI가 발췌하기 좋은 형태인지 점수화합니다. 측정: 본문 텍스트 정규식 + 패턴 매칭으로 5신호 비율 계산 후 가중 평균.

⑩ CEP 장면 점유 (cepScene) ⭐ — 가중치 6%

본 플랫폼의 또 다른 자체 차별점입니다. CEP(Category Entry Point - 카테고리 진입 순간) 이론에 따라 "사람"이 아닌 "순간(scene)"을 표적으로 한 콘텐츠 비율을 측정합니다. "아침에 갑자기 ○○할 때", "야근 후 ○○가 필요할 때" 같은 장면 표적이 많을수록 AI 답변에 등장할 확률이 올라갑니다. 측정: 콘텐츠 내 "○○할 때/순간/상황" 패턴 검출 비율.

4. AI 인용 글쓰기 6원칙 ⭐

Perplexity·SearchGPT·AI Overview 같은 LLM이 답변에 발췌하기 좋은 콘텐츠의 6가지 원칙입니다. KPI ⑨ AI 인용 5신호 (aiCitation)는 이 원칙의 5개 측정 신호로 직접 산출됩니다.

💡 핵심 공식: AI 인용 = EEAT × 데이터량 × 질문형 × 반복성 × 외부신호 × CTA
곱셈이라 어느 한 요소가 0이면 AI 인용 가능성도 0입니다.

📜 6대 원칙

  1. 원칙 0. CEP 장면 점유 — "사람"이 아니라 "순간"을 표적으로. ("아침에 갑자기 ○○할 때")
  2. 원칙 1. 작성자 권위 — 도입부와 결론에 작성자/대표 정보를 별도 단락으로 명시.
  3. 원칙 2. 정의문 — 모든 H2 첫 문장은 "X는 ~이다" 형태로 시작 (LLM 추출 친화).
  4. 원칙 3. 구조화 — 번호 리스트, 표, Q:A: 페어, 명시적 수치(%, 원, 명, 건).
  5. 원칙 4. AI 인용 4신호 ⭐ — 질문형 H2 + 브랜드 반복 + 외부 신호 + CTA 도달률 (아래 표).
  6. 원칙 5. 행동 유도 — 핵심 답 + CTA 솔루션 + TL;DR.

📊 측정 5신호 — KPI ⑨ aiCitation 직접 산출

다음 5가지 신호는 입력 콘텐츠에서 정량적으로 측정되어 KPI ⑨ AI 인용 5신호 (aiCitation, 가중치 10%) 점수에 직접 반영됩니다. (ai_writing 4 측정 신호 + 원칙 2 정의문 H2)

신호 목표 측정 방법
① 질문형 H2 비율 ≥ 50% H2 중 "?", "어떻게/왜/언제/무엇/어디/어떤" 포함 비율
② 정의문 H2 비율 (원칙 2) ≥ 50% H2 첫 문장이 "X는 ~이다/입니다" 정의문 패턴인 비율
③ 브랜드 반복 비율 ≥ 50% 각 H2 섹션에 브랜드명이 등장하는 비율
④ 외부 신호 비율 ≥ 30% 후기 인용 + 언론 보도 + "에 따르면" 패턴 충족도
⑤ CTA 도달률 ≥ 50% 800자 블록당 CTA 키워드(상담/예약/메신저) 등장 비율

🦴 7단계 골격 (의무)

  1. 도입부 — 브랜드 정의 + 핵심 가치 1~2문장
  2. H2 5~7개 — 그 중 5개 이상 질문형으로
  3. 단계별 프로세스 — 번호 리스트 (1. 2. 3.)
  4. 사례·수치 — 구체적 숫자 5개 이상 + Before/After
  5. FAQ Q1~Q5+ — 각 Q는 질문형, A는 정의문 시작
  6. 결론 + CTA — 메신저/전화/예약 3종 동시
  7. 비교 표 3열 — | 항목 | 업계 평균 | 우리 |

📏 분량 권장

  • 짧은 답변: 1,200~1,500자
  • 표준 GEO 콘텐츠: 1,800~2,200자 ⭐ (권장)
  • 깊이 있는 가이드: 2,200~2,500자

🌳 결정 트리 (콘텐츠 점검)

타겟이 사람? → 장면으로 변환
브랜드 1회? → 2회 이상 반복
설명형 H2? → 질문형으로 변경
짧은 결론 X? → 각 H2 끝 "핵심:" 한 줄
케이스 X? → Before/After 4단 구조
외부 인용 X? → 후기·언론·논문 키워드 추가
CTA X? → "[X 문제]는 [브랜드]로 해결" 패턴
1글로 끝? → 30개 질문형 파생 추가
1500자 미만? → 1800자+ 보강
🚀 한 글로 30개 파생: 마스터 글 1개를 그대로 두고, 30개 질문형 파생을 만들어 AI 인용량을 늘립니다. /90upgrade 스킬 또는 ✨ 90점 글 메뉴에서 자동 실행됩니다.

5. 등급 시스템 (6단계)

10개 KPI 가중치 평균(합 100%)으로 종합 점수(0~100)가 산출되며, 6단계 등급으로 분류됩니다.

👑 A+ Premium (95~100) · 운영 유지 (월 50~150만원)
ChatGPT·Claude·Perplexity 등 모든 AI 검색에서 우선 인용·추천되는 시장 지배 수준입니다. 외부 인프라(AI 봇 접근·사이트 지도·검색 색인)·구조화 데이터·콘텐츠 깊이·외부 권위가 모두 충족되어 경쟁사가 따라잡기 어렵습니다. 추가 개선보다 모니터링과 미세 조정을 통한 우위 유지에 집중합니다.
💪 A 우수 (85~94) · 콘텐츠 강화 (월 50~150만원)
AI 검색 결과에서 안정적으로 발견되며 인용·추천 빈도가 상위권에 속합니다. 핵심 기술 신호(AI 봇 접근·구조화 데이터·E-E-A-T)는 모두 갖춰진 상태로 약점 KPI 한두 개만 추가 강화하면 1위권 도달이 가능합니다. 콘텐츠 발행 가속과 외부 백링크 확보가 다음 단계 핵심입니다.
📈 B 보통 (70~84) · 부분 개선 (200~500만원, 2~4주)
검색 노출 기반은 형성되어 있으나 AI 인용·추천 최적화 측면에서는 미흡합니다. 일반 검색에서는 발견되지만 ChatGPT·Perplexity 답변에 등장할 가능성은 낮습니다. 약점 KPI 2~3개에 집중 투자하면 상위권 진입이 가능합니다. 구조화 데이터와 콘텐츠 깊이 보강이 우선 과제입니다.
⚠️ C 미흡 (55~69) · 전면 개선 또는 신규 개발 비교 (200~400만원)
AI 검색 시대 핵심 신호가 부족해 구조 정비가 시급한 단계입니다. 외부 인프라(사이트 지도·검색 색인)·구조화 데이터·콘텐츠 깊이 중 다수가 임계점 미만이며 부분 개선만으로 한계가 있을 수 있어 종합적 재설계 검토가 필요합니다. 현재 시스템 수정 가능성을 먼저 확인합니다.
🚨 D 부족 (40~54) · 신규 개발 권장 (400만원, 3주)
AI 검색에서 인용·추천이 거의 어려운 상태입니다. 약점 KPI가 다수로 부분 개선보다 신규 개발이 효율적이며 임대형 시스템이라면 운영자가 자유롭게 콘텐츠를 추가·수정할 수 있는 자체 도메인 구조로 전환하는 것이 우선입니다. 도메인은 유지, 시스템만 교체합니다.
🚨 F 위급 (0~39) · 신규 개발 필수 (400만원, 3주, 즉시)
AI 검색에서 사실상 발견되지 않는 위급 상태입니다. AI 봇 차단·사이트 지도 부재·색인 누락 등 구조적 결함이 누적되어 기존 구조로는 회복이 어렵습니다. 신규 개발이 필수이며 도메인은 유지하되 시스템 전체를 AI 친화적으로 교체해야 경쟁 진입이 가능합니다.

6. 진단 워크플로

URL 또는 직접 콘텐츠 입력 한 번으로 8단계 자동 진단이 실행됩니다.

  1. 정보 입력 — URL 또는 직접 콘텐츠 텍스트를 입력합니다. 업종은 자동 감지되거나 수동 선택 가능.
  2. 자동 크롤링 — 메인 페이지 HTML + /robots.txt + /sitemap.xml을 병렬로 가져옵니다.
  3. 외부 측정 — 검색 색인된 페이지 수와 백링크 휴리스틱을 외부 신호 기반으로 추정합니다.
  4. 구조 분석 — Schema.org JSON-LD, FAQ 마크업, 메타 태그, OG 이미지를 검출·정규화합니다.
  5. Gemini AI 분석 — 10대 KPI 점수를 산출하고, 가중치(합 100%)로 가중 평균 종합 점수를 계산합니다.
  6. 페널티 매트릭스 적용 — 구조화 데이터 부재 시 -15%, AI 봇 차단 시 -30% 등 임계 페널티를 적용합니다.
  7. AXOS 온톨로지 인과 분석 — 약점 KPI들의 근본 원인(root cause)을 8종 노드 + 9종 관계 그래프로 추적합니다.
  8. 솔루션 + 권장 경로 — 5단계 권장 경로(A+ 운영 유지부터 F 신규 개발 필수까지)와 견적을 자동 매칭합니다.
💡 병렬 크롤링: 메인 페이지 + robots.txt + sitemap.xml + 외부 색인을 동시에 가져와 30초 내로 진단을 완료합니다.

7. GEO-AIO 솔루션 연결

진단 등급에 따라 5단계 권장 경로가 자동 매칭됩니다. 등급별로 부분 개선·전면 개선·신규 개발 중 가장 효율적인 옵션을 제시합니다.

등급 권장 경로 견적 / 기간
A+ Premium (95~100)운영 유지 — 모니터링 + 미세 조정월 50~150만원
A 우수 (85~94)콘텐츠 강화 — 약점 KPI 1~2개 보강월 50~150만원
B 보통 (70~84)부분 개선 — 약점 KPI 2~3개 집중 투자200~500만원, 2~4주
C 미흡 (55~69)전면 개선 vs 신규 개발 비교 — 현행 시스템 수정 가능성 먼저 진단200~400만원
D 부족 (40~54)신규 개발 권장 — 도메인 유지, 시스템 교체400만원, 3주
F 위급 (0~39)신규 개발 필수 — 즉시 착수400만원, 3주, 즉시
💡 도메인은 유지: 신규 개발 권장 등급(D, F)에서도 기존 도메인은 그대로 유지합니다. 시스템 구조만 AI 친화적으로 교체하므로 SEO 자산 손실이 없습니다.

8. API 문서

모든 API는 Vercel Edge에서 동작하며 JSON 요청·응답을 사용합니다. 응답에는 새 10대 KPI 키가 포함됩니다.

POST /api/analyze

진단 분석을 실행합니다. 새 KPI 응답 형식.

// Request
{
  "companyName": "디지털스마일치과",
  "websiteUrl": "https://example.com",
  "industry": "dental",
  "rawContent": "(선택) 직접 콘텐츠 텍스트"
}

// Response (새 KPI 형식)
{
  "totalScore": 78,
  "grade": "B",
  "scores": {
    "botAccess": 95, "sitemapStatus": 80, "indexExposure": 70,
    "structuredData": 60, "pageInfo": 85, "contentDepth": 72,
    "externalAuthority": 55, "eeat": 68,
    "aiCitation": 74, "cepScene": 40
  },
  "rationale": { ... },
  "penalties": [{ "key": "structuredData", "amount": -15 }]
}

POST /api/recommend

진단 점수 기반 5단계 권장 경로와 견적을 반환합니다.

{
  "scores": { ... },
  "totalScore": 42,
  "grade": "D",
  "companyName": "...",
  "industry": "..."
}

POST /api/chat

진단 컨텍스트 기반 AI 상담사 대화.

{
  "message": "AI 봇 접근 점수를 어떻게 올리나요?",
  "history": [],
  "diagnosisContext": { /* 진단 결과 */ }
}

POST /api/cep-discover

CEP(Category Entry Point) 5단계 발굴을 실행합니다. 업종별 "순간(scene)"을 자동 발굴해 콘텐츠 표적을 제시합니다.

{
  "industry": "dental",
  "brand": "디지털스마일치과",
  "region": "강남"
}

🆕 POST /api/rewrite-content

90점 글 자동 재작성. 입력 글의 부족 신호를 자동 보강해 GEO Score 90점 이상으로 재작성합니다. 자가 검증 루프 최대 3회.

{
  "originalContent": "...",
  "brand": "디지털스마일치과",
  "industry": "dental",
  "targetScore": 90
}

🆕 POST /api/derive-30

마스터 글 1개를 입력하면 30개 질문형 파생을 자동 생성합니다. AI 인용량 확장용.

{
  "masterContent": "...",
  "brand": "...",
  "count": 30
}

🆕 POST /api/citation-test

AI 인용성 실제 검증. 콘텐츠를 Perplexity·SearchGPT·Claude·Gemini 등에 직접 질의해 실제 인용 여부를 확인합니다.

{
  "content": "...",
  "queries": ["○○ 추천", "○○ 비용"],
  "engines": ["perplexity", "searchgpt", "claude", "gemini"]
}

🆕 POST /api/generate-report

영업용 단일 HTML 리포트 생성. AX Biz Group 브랜드의 단일 HTML 파일로 진단 결과 + 솔루션 + 견적을 통합 출력합니다.

{
  "diagnosis": { ... },
  "clientName": "...",
  "salesperson": "심재우"
}

GET /api/health

서비스 상태와 환경 변수(GEMINI_API_KEY 등) 설정 여부를 반환합니다.

9. 통합 페이지 / 영업 도구

분산된 결과 화면을 한 번에 보거나 영업·제안 단계로 바로 연결할 수 있는 통합 페이지를 제공합니다.

📋 통합 결과 페이지 (result-tabs.html)

8개 결과 탭 - 개요(overview)·KPI 상세·문제·솔루션·재작성(rewrite)·CEP·온톨로지·비교 - 을 한 페이지에서 탭 전환으로 모두 확인할 수 있는 통합 뷰입니다. 진단 후 결과를 빠르게 훑어보거나 영업 미팅에서 화면 전환 없이 한 흐름으로 설명할 때 사용합니다.

  • 접근: 상단 내비게이션 "📋 통합 결과" 메뉴
  • 탭 구성: overview / kpi / problems / solution / rewrite / cep / ontology / comparison
  • 활용: 진단 직후 + 영업 프레젠테이션

🎁 영업 리포트 생성기 (generate-report.html)

AX Biz Group 영업팀이 고객에게 전달할 단일 HTML 리포트 파일을 자동 생성합니다. 진단 결과 + 권장 경로 + 견적 + 솔루션 매트릭스가 1개 HTML 파일로 패키징되어 이메일·슬랙으로 즉시 전달 가능합니다.

  • API: POST /api/generate-report
  • 출력: 인쇄·PDF 변환 가능한 단일 HTML 파일
  • 브랜딩: AX Biz Group 로고·컬러 적용

📋 AI최적화 홈페이지 개발 제안서 (proposal.html)

D·F 등급 진단 후 신규 개발이 권장된 고객에게 즉시 제시할 수 있는 AI최적화 홈페이지 개발 제안서 템플릿입니다. 5단계 권장 경로·견적·기간·산출물이 미리 정리되어 있습니다.

  • 경로: 푸터 "📋 AI최적화 홈페이지 개발 제안서" 링크
  • 구성: 진단 요약 → 문제 정의 → 솔루션 → 일정 → 견적 → CTA

10. 자주 묻는 질문 (FAQ)

Q. 왜 GEO가 SEO보다 중요한가요?
사용자가 검색 결과를 클릭하지 않고 AI 답변만 보는 시대로 넘어가고 있기 때문입니다. AI가 인용하지 않으면 사실상 존재하지 않는 것과 같습니다.
Q. 진단은 얼마나 걸리나요?
약 30초~1분 내 완료됩니다. URL 크롤링 + Gemini AI 분석 시간이 포함됩니다.
Q. 진단 결과는 어떻게 저장되나요?
진단 결과는 브라우저의 localStorage에 저장됩니다. 대시보드 페이지에서 최대 50건까지 이력 확인이 가능합니다.
Q. 어떤 업종에 적용 가능한가요?
치과, 병원, 법무, 교육, 미용, 음식점, 유통, B2B 등 거의 모든 업종에 적용됩니다.
Q. 점수가 낮으면 어떻게 해야 하나요?
진단 결과의 "솔루션 제안" 탭에서 우선순위 액션과 추천 패키지를 확인하세요. 가장 빠른 해결책은 GEO-AIO 솔루션 도입입니다.
Q. Gemini API 키가 없어도 사용 가능한가요?
가능합니다. API 키가 없으면 휴리스틱(규칙 기반) 점수가 제공되며, 개인화 메시지 일부 기능은 제한됩니다.
Q. 결과 페이지가 8개로 나뉘어 있는데 한번에 보고 싶어요
상단 내비게이션의 "📋 통합 결과" 메뉴(result-tabs.html)를 사용하세요. 개요·KPI·문제·솔루션·재작성·CEP·온톨로지·비교 8개 탭을 한 페이지에서 전환하며 확인할 수 있습니다. 영업 프레젠테이션이나 빠른 훑어보기에 최적화되어 있습니다.
Q. 영업용 리포트는 어떻게 만드나요?
영업 리포트 생성 페이지(generate-report.html)에서 진단 결과를 입력하면 AX Biz Group 브랜드의 단일 HTML 파일로 리포트가 자동 생성됩니다. 이메일이나 슬랙으로 즉시 전달 가능하고, 인쇄·PDF 변환도 지원합니다. API 엔드포인트는 POST /api/generate-report입니다.
Q. AI 인용성을 실제로 검증할 수 있나요?
가능합니다. "🔍 인용성 검증" 메뉴(citation-test.html)에서 콘텐츠를 입력하면 Perplexity·SearchGPT·Claude·Gemini 같은 실제 AI 검색에 자동으로 질의를 보내 인용 여부를 검증합니다. API는 POST /api/citation-test이며, 점수 예측이 아니라 실제 결과 기반 측정이 특징입니다.
