## 1. 플랫폼 개요

### 핵심 정의

**GEO Score AI**는 가망 고객·기존 고객의 **홈페이지/블로그를 진단·분석**하여, AI 검색(ChatGPT·Claude·Perplexity·Gemini)에서 **인용·추천될 가능성**을 10가지 KPI로 점수화하고, **부분 개선 또는 신규 개발** 의뢰로 자연스럽게 연결하는 **세일즈 퍼널 엔진**입니다.

### 한 줄 정의

> "AI 검색에서 귀사와 홈페이지는 존재하고 인용됩니까?"
> 이 질문에 **0~100점**으로 답하고, **A+~F 6단계**로 등급화하여 사용자에게 **즉각 행동을 유발**하는 진단 플랫폼.

### 3가지 핵심 가치

| 가치 | 설명 |
|---|---|
| **결정적 점수** | 의사난수가 아닌 **신호 기반 결정적 함수**. 같은 입력 → 항상 같은 점수 |
| **다층 측정** | 외부 인프라(4) + 콘텐츠(10 KPI) + ai_writing(5신호) **3계층 통합** |
| **세일즈 직결** | 점수 → 등급 → 권장 경로 → 견적까지 **자동 산출**, 가망 고객의 의뢰 동기 발생 |

### 핵심 철학

> **"문제를 보여주고, 해결을 유일하게 만들고, 계약으로 연결한다"**

단순 분석 툴이 아니라, **진단 → 점수화 → 문제 인식 → 해결 제안 → 계약 전환**까지 이어지는 완전 자동화 파이프라인.

## 2. GEO 진단의 필요성

### AI 검색 시대 패러다임 변화

```
[과거 SEO 시대]                [현재 GEO 시대]
사용자 → 검색엔진 → 클릭     사용자 → AI 답변 (클릭 X)
                              ↓
                              AI가 추천한 1~3개만 발견
```

**핵심 변화**: 사용자가 검색 결과를 **클릭하지 않고 AI 답변만 보는 시대**. AI가 인용하지 않으면 사실상 **존재하지 않는 것**과 같음.

### GEO ≠ SEO — 5가지 결정적 차이

| 측면 | SEO | GEO |
|---|---|---|
| **목표** | 검색 결과 노출 | AI 답변에 인용 |
| **신호** | 키워드, 백링크 | 구조화 데이터, FAQ, AI 봇 접근 |
| **포맷** | 긴 글 | 정의문 H2, 질문형 H2, FAQ |
| **권위** | 도메인 권위(DA) | E-E-A-T + 외부 신호 |
| **결과** | 트래픽 | LLM 학습 + 답변 인용 |

### 가망 고객이 직면한 위험

1. **AI 봇 차단**: robots.txt에서 GPTBot/ClaudeBot 등 7봇 모두 차단 → AI 학습 불가능
2. **Sitemap 손상 또는 부재**: 검색·AI 봇이 사이트 구조 파악 못 함
3. **색인 부족**: 구글 색인이 5건 미만 → AI 발견 자체 어려움
4. **Schema 미적용**: AI는 구조화된 정보를 우선 인용
5. **임대형 시스템**: 운영자가 자유롭게 콘텐츠 추가·수정 불가능

→ 이 5가지 중 **3개 이상이 약점**이면 **AI 검색에서 사실상 부재** 상태.

### GEO 진단의 효과 (3개월 추적)

- **F 위급(0~39점)**: AI 인용 0% → 신규 개발 후 **70+ A 우수** 도달 가능
- **D 부족(40~54점)**: 부분 개선 + 콘텐츠 발행으로 **60~70점** 진입
- **C 미흡(55~69점)**: Schema·FAQ 적용 + ai_writing 5신호 충족 시 **85+** 가능
- **B 보통(70~84점)**: 외부 백링크 캠페인으로 **A+ Premium** 도달

## 3. 차별성 — 다른 진단 도구와의 차이

### vs SEO 진단 도구 (Ahrefs, SEMrush 등)

| 측면 | 일반 SEO 도구 | GEO Score AI |
|---|---|---|
| 측정 | 검색 순위·트래픽·키워드 | **AI 인용 가능성** 직접 측정 |
| 항목 | 200+ 다중 지표 | **10 핵심 KPI** (가중치 합 100%) |
| 결과 | 데이터 분석 리포트 | **자동 견적 + 권장 경로** |
| 가격 | 월 $99~$999 | **무료 진단 + 자동 영업 리포트** |

### vs AI 글쓰기 도구 (ChatGPT, Jasper 등)

| 측면 | 일반 AI 글쓰기 | GEO Score AI |
|---|---|---|
| 목표 | 자연스러운 글 | **측정 가능한 점수** |
| 검증 | 사람의 주관 평가 | **결정적 점수 함수** |
| 반복 | 사용자가 수동 | **자가 검증 루프 (자동)** |
| 결과 | 텍스트만 | **점수 비교 + KPI 차트 + 좌우 비교** |
| 재현성 | 낮음 (매번 다름) | **높음 (같은 입력 → 같은 점수)** |
| 도메인 | 범용 | **GEO/AIO 최적화 특화** |

### 자체 차별점 (이 플랫폼만 보유)

1. **ai_writing 5신호 측정** — 질문형 H2 / 정의문 H2 / 브랜드 반복 / 외부 신호 / CTA 도달률 (KPI 9 직접 산출)
2. **CEP 장면 점유** — 결정적 순간(Category Entry Point) 발굴 + 매핑 (KPI 10)
3. **AXOS 온톨로지** — 8 노드 + 9 관계로 KPI 인과 관계 시각화 + Root Cause 자동 추적
4. **90upgrade 자가 검증 루프** — Gemini 재작성 → 점수 검증 → 90점 미달 시 부족 영역 강조 후 재시도 (최대 3회)
5. **30개 질문형 파생 자동 생성** — 1글 → 30파생으로 데이터량 부족 한 번에 해결
6. **AI 인용성 실제 검증** — 10개 가상 질문에 대한 답변에서 brand 인용 횟수 측정 (KPI 점수가 진짜 인용으로 이어지는지 검증)
7. **AX Biz Group 영업 리포트 양식** — 다크 네이비 + 골드 + 아이보리 3탭 단일 HTML 자동 생성

## 4. 10대 KPI 체계

가중치 합 **100%** — AX Biz Group 영업 리포트 표준 8 + 자체 차별점 2

| # | id | 한글명 | 가중치 | 측정 방법 | 50단어 핵심 |
|---|---|---|---:|---|---|
| ① | botAccess | 🤖 AI 봇 접근 | **14%** | robots.txt에서 GPTBot·ClaudeBot·ChatGPT-User·PerplexityBot·Google-Extended·Amazonbot·CCBot 7종 차단 검출 | ChatGPT·Claude·Perplexity 등 7종 AI 봇이 사이트를 크롤링할 수 있는지 측정. 차단 시 LLM 학습 자체가 불가능해 인용·추천에서 사라짐. 5종 이상 차단 시 신규 개발이 필수적인 결정적 약점. |
| ② | sitemapStatus | 🗺️ Sitemap 상태 | **10%** | sitemap.xml URL 수 + lastmod | sitemap.xml이 정상 작동하고 충분한 URL이 등록되어 있는지 측정. 검색엔진과 AI 봇이 사이트 구조를 효율적으로 파악하는 핵심 인프라이며, 손상되거나 외부 도메인을 가리키면 색인 효율이 급격히 떨어짐. 자기 도메인 + 50개 이상 URL이 이상적. |
| ③ | indexExposure | 🔎 검색 색인 | **13%** | 색인 추정 (sitemap × 0.6 + 내부링크 × 1.2) | 구글·네이버에 색인된 페이지 수를 측정. 색인 수가 많을수록 AI 검색 결과에 발견될 가능성이 높아지며, 5건 미만일 경우 사실상 AI 인용 자체가 어려워짐. sitemap·내부 링크 다양성·콘텐츠 신선도가 색인 수의 핵심 결정 요인. |
| ④ | structuredData | 📐 구조화 데이터 | **12%** | Schema.org JSON-LD · FAQPage · Organization 마크업 | Schema.org JSON-LD·FAQPage 등 구조화 마크업 적용도를 측정. AI는 구조화된 정보를 우선 인용하므로, FAQPage·Organization·Article Schema가 적용된 사이트가 답변에 자주 발췌됨. 미적용 시 다른 KPI에도 -15% 페널티가 적용되는 결정적 영역. |
| ⑤ | pageInfo | 📄 페이지 정보 | **8%** | title · description · canonical · OG · H1/H2 | title·description·canonical·og 태그·H1/H2 헤딩 구조의 완비도를 측정. AI가 페이지의 주제와 위계를 정확히 파악하는 1차 정보원으로, 페이지마다 고유한 메타 정보가 있어야 검색·인용 정확도가 높아짐. 단일 메타로 통일된 사이트는 점수가 낮음. |
| ⑥ | contentDepth | 📚 콘텐츠 깊이 | **10%** | 블로그 발행 빈도 + 본문 양 + 최신성 | 블로그 발행 빈도·본문 양·최신성을 종합 측정. AI 학습 데이터는 콘텐츠의 양과 다양성에 비례하므로, 30일 주기 발행 + 1500자 이상 본문이 이상적. 단발성 페이지만 있는 사이트는 AI에 학습될 기회가 부족해 인용에서 누락됨. |
| ⑦ | externalAuthority | 🌐 외부 권위 | **9%** | 백링크 추정 (외부링크 × 0.6 + 언론키워드 × 1.5) | 백링크·언론 보도·외부 매체 언급 수를 측정. AI는 제3자 신호가 풍부한 출처를 선호하므로, 보도자료 발신·블로거 협업·전문 매체 등재가 인용 가중치를 높임. 자체 사이트 콘텐츠만 있고 외부 인용이 0건이면 AI 신뢰도가 매우 낮음. |
| ⑧ | eeat | 🏆 E-E-A-T 신호 | **8%** | 작성자·자격·연혁·연락처 | 작성자·자격·연혁·연락처 등 신뢰 신호 4축을 측정. 구글이 정의한 Experience·Expertise·Authority·Trust 프레임워크로, 의료·금융·법률 등 YMYL 분야에서 특히 중요. 대표 프로필·자격증·운영 연혁·실명 후기를 노출해 AI 신뢰 가중치를 확보. |
| ⑨ | aiCitation | ⭐ AI 인용 5신호 | **10%** ⭐ | ai_writing 5신호 (질문형/정의문 H2·브랜드 반복·외부 신호·CTA 도달률) | 질문형 H2·정의문 H2·브랜드 반복·외부 신호·CTA 도달률 5가지 ai_writing 측정 신호를 측정. 5신호 모두 50% 이상 충족 시 AI 답변에 인용될 가능성이 비약적으로 상승하며, /ai_writing 6원칙 기반 자체 차별점. KPI 4의 직접 산출 지표. |
| ⑩ | cepScene | 🎯 CEP 장면 점유 | **6%** ⭐ | "순간/장면" 콘텐츠 검출 | 사용자의 결정적 순간(Category Entry Point)을 표적으로 한 장면형 콘텐츠 보유 정도를 측정. "아침에 갑자기 ○○할 때" 같은 구체적 상황을 다룬 콘텐츠가 풍부할수록 AI가 실사용자 질문에 매칭되어 인용될 가능성이 높아짐. 자체 차별점. |

## 5. 6단계 등급 시스템

### 영업 사각지대 회피 (점수 엄격화 적용)

이전 60점대(B 보통)에 평범한 사이트가 머무르며 "괜찮네"라고 인식되는 영업 사각지대 문제를 다음 4가지로 해결:

## 9. ai_writing 6원칙 + 5신호

## 12. 신규 기능 7종

### 12.1. ✨ 90점 글 자동 생성 (`/90upgrade`)

### 12.2. 🌳 30개 질문형 파생 (`/api/derive-30`)

- 입력: 마스터 글 또는 CEP 장면 + 브랜드
- 처리: Gemini로 30개 질문형 파생 자동 생성 (6 카테고리 균형: 비용/절차/비교/장단점/추천/사후관리)
- 출력: 카드 그리드 + 일괄 다운로드(JSON/MD/PDF) + **30개 모두 90점 글로 일괄 변환** 옵션

### 12.3. 🔍 AI 인용성 실제 검증 (`/api/citation-test`)

- 입력: 진단된 콘텐츠 + 브랜드
- 처리:
  1. Gemini로 10개 가상 사용자 질문 생성 (general 5 + branded 5)
  2. 각 질문에 대해 콘텐츠를 출처로 답변 시뮬레이션
  3. 답변에서 brand 인용 횟수 측정
- 출력: **인용률 % + 등급 + 질문별 답변 결과**
- 효과: KPI 점수가 **진짜 인용**으로 이어지는지 결정적 검증

### 12.4. 🎯 CEP 발굴 (`/api/cep-discover`)

5단계 워크플로:
1. **표현 추출** — 일상 표현 8~12개
2. **클러스터화** — 4~5개 클러스터로 그룹화
3. **장면 정의** — CEP 장면 문장 도출
4. **점수화** — Market×BrandFit×Evidence 30점 만점
5. **최종 좌표** — 메시지·제품 포인트·콘텐츠 아이디어·GEO 근거

### 12.5. 🎁 영업용 단일 HTML 리포트 (`/api/generate-report`)

  1. **진단 리포트** (와인색) — 8 KPI 별점 + 핵심 결함 + 강점
  2. **신규 개발 제안** (에메랄드) — 권장 경로 + 4 Phase + 견적 (400만원/3주)
  3. **대표 프로필** (골드) — AX Biz Group 심재우 컨설턴트
- 출력: **57KB 단일 HTML 다운로드** — 의사결정자에게 즉시 제출 가능

## 16. FAQ

### Q. 왜 GEO가 SEO보다 중요한가요?

사용자가 검색 결과를 클릭하지 않고 AI 답변만 보는 시대로 넘어가고 있기 때문입니다. AI가 인용하지 않으면 사실상 존재하지 않는 것과 같습니다.

### Q. 진단은 얼마나 걸리나요?

약 30초~1분 내 완료됩니다. URL 크롤링 + 외부 인프라 측정 + Gemini AI 분석 시간이 포함됩니다.

### Q. 진단 결과는 어떻게 저장되나요?

진단 결과는 브라우저의 localStorage에 저장됩니다. 대시보드 페이지에서 최대 50건까지 이력 확인이 가능합니다.

### Q. 어떤 업종에 적용 가능한가요?

치과, 병원, 법무, 교육, 미용, 음식점, 유통, B2B 등 거의 모든 업종에 적용됩니다. 업종별 가중치는 자동 보정됩니다.

### Q. 점수가 낮으면 어떻게 해야 하나요?

진단 결과의 "솔루션 제안" 탭에서 우선순위 액션과 추천 패키지를 확인하세요. 점수 구간에 따라 콘텐츠 강화 / 부분 개선 / 신규 개발이 자동 권장됩니다.

### Q. Gemini API 키가 없어도 사용 가능한가요?

가능합니다. API 키가 없으면 휴리스틱(규칙 기반) 점수가 제공됩니다. 정밀 진단을 위해 GEMINI_API_KEY 설정을 권장합니다.

### Q. 결과 페이지가 8개로 나뉘어 있는데 한번에 보고 싶다

`/result-tabs.html?id=<진단ID>` 통합 페이지에서 8개 탭 콘텐츠를 한 화면에 펼쳐 볼 수 있습니다. 각 결과 페이지의 "📋 8개 분석 결과를 한 페이지에서 모두 보기" CTA로 진입 가능합니다.

### Q. 영업용 리포트 만들려면?

`/generate-report.html?id=<진단ID>` 페이지에서 진단 결과를 AX Biz Group 영업 리포트 양식 단일 HTML(57KB)로 자동 변환합니다. A4 인쇄 친화 디자인으로 의사결정자에게 즉시 제출 가능합니다.

### Q. AI 인용성을 실제로 검증할 수 있나?

`/citation-test.html` 페이지에서 입력한 콘텐츠가 AI 검색 답변에 인용될 가능성을 10개 가상 질문으로 시뮬레이션합니다. KPI 점수가 진짜 인용으로 이어지는지 결정적 검증이 가능합니다.
